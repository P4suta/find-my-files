# ADR-0019: 絞り込みモード(focused search)はUI層の純粋なクエリ書き換え

日付: 2026-06-12 / 状態: 採用済み

## 決定

「一般ユーザーが探すファイルはディレクトリも形式も限られている」要望に対し、絞り込みモードを
**UI層の純粋なクエリ書き換え**(`ViewModels/FocusedQueryRewriter.Compose` — static・副作用なし)で
実現する。エンジン(Rust)・ワイヤ契約・インデックスには一切触れない。

- ユーザークエリをトップレベル `|` で分割(引用符内の `|` は分割しない — エンジンの
  tokenizer と同じ引用符規則)し、各ORグループに設定由来の suffix を付加して再結合:
  - 除外パス: 各エントリ p につき `!path:"p"`(引用符付き否定 — `\windows\` 等のノイズ領域)
  - 形式ホワイトリスト: `ext:e1;e2;…` **1項**(`ext:` の値はOR意味論なので項を増やさない)
- **衝突回避(ユーザーの明示意図が常に勝つ)**: グループが `ext:`/`regex:` を含めば ext
  ホワイトリストを、`path:` または `\` を含めば除外パスを付加しない。判定はグループ文字列の
  単純な substring 検査(過剰一致は「付加をスキップする」方向にしか倒れない=安全側)
- 空クエリは空のまま返す(「空クエリはエンジンに投げない」規則は Orchestrator の所有のまま)。
  `"` を含む除外パス設定値はクエリ言語でエスケープ不能なため無視+warn(初回のみ)
- 設定は `%APPDATA%\find-my-files\settings.json`(UI所有): `focused_search`(既定 **true**)/
  `focused_exclude_paths` / `focused_extensions`。UIは検索ボックス隣の ToggleButton、
  トグル変更はフィルタ起因の再クエリ(`RequeryOrigin.Filter` = 先頭リセット)
- `SearchOrchestrator.FocusedSearch` の既定は **false**(既存テスト・既存挙動を無傷に保つ)。
  プロダクト配線(MainViewModel)だけが settings の true を流し込む

## 根拠

- **既存クエリ言語で全部表現できる**: `ext:` は1項OR(`;`区切り)、`path:` は引用符対応+`!`否定、
  `|` はORグループ(fmf-core/src/query/ast.rs)。新しい演算子・新しいフィルタ機構が不要
- **残余コストはヒット数比例**: エンジンの線形スイープは変わらず、付加項は候補を減らす方向に
  しか働かない。書き換え自体は文字列結合(キーストローク毎・数µs)で遅延予算に乗らない
- **エンジン非接触 = perf-gate 不要**: fmf-core を触らないため、昇格ベンチ・回帰ゲートの儀式
  なしで出荷できる。ワイヤバイト・契約・ゴールデンコーパスも不変

## 却下案

- **エンジン内プリセットフィルタ(インデックスビット)**: エントリ毎に「ノイズ領域フラグ」を
  事前計算すれば残余コストはほぼゼロになるが、インデックスレイアウト変更(RAM予算・
  スナップショット版数)+設定変更時の全再計算+ UI設定とサービス所有インデックスの所有権交差が
  発生する。書き換え方式の実測が遅くなってから検討する最適化であって、先に払うコストではない
- **ランキング(関連度順ソート)**: 「ズバリ数件」の正攻法はフィルタでなくスコアリングだが、
  スコアリング基盤(使用頻度・最近性・パス深度等の特徴量と順列キャッシュ)が丸ごと必要で、
  現行の遅延ソート順列(ADR-0006)と直交しない。**将来課題として明記** — フィルタで足りる間は
  作らない

## 影響

- 変更面: `FocusedQueryRewriter`(新規)+ `AppSettings` 3キー + `SearchOrchestrator` の
  書き換え1点 + MainPage の ToggleButton。エンジン・contract/golden・Generated は不変
- F12パネル/ログのクエリ表記は**書き換え後**(エンジンが実際に見た文字列)— 調査時は
  settings.json の2リストを念頭に置くこと
- 既定ONのため「ファイルがあるはずなのに出ない」問い合わせはまずトグルOFFで切り分ける
  (ツールチップに除外の存在を明記済み)

## 再検討トリガ

- 絞り込みONで検索 p99 > 50ms(性能合格ライン超過 — エンジン内プリセットフィルタを再評価)
- フィルタでは「ズバリ度」が不足し、ランキング(スコアリング基盤)が必要になったとき
